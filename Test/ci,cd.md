# CI/CD

## CI (Continuous Integration)

- 지속적인 통합
- 코드 변경사항을 주기적으로 빈번하게 머지해야 함
  - 코드의 변경사항은 테스트코드를 포함한 것을 말함
- 테스트코드가 있어야 통합을 위한 단계(빌드, 테스트, 머지)가 자동화가 될 수 있음

### CI를 통해 얻을 수 있는 장점

- 코드의 퀄리티 향상
- 개발 생산성 향상
- 버그 수정 용이
- 문제점을 빠르게 발견

## CD (Continuous Depolyment)

- 지속적인 배포
- CI가 원활해야 CD도 원활하게 이루어짐
